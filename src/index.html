Work History

C.H. Robinson // 2016 -> Present

I started at C.H. Robinson working on the Compensation delivery team. There, we used React/Redux, .NET Service Stack APIs, and SQL Server to create a compensation planning tool that managers could use to plan their employess compensation structure.

After a couple of years, the Compensation team was dispersed to other teams in the org. I joined the Orders delivery team. There, I was responsible with managing change into our Orders module inside of a product called Navisphere. I worked with older technologies like WinForms, WPF, and still Service Stack APIs, but also newer front end technologies like Angular, TypeScript, JavaScript, and build techologies using NodeJS like Webpack and Jest. This was before things like create-react-app were around.

After another couple of years, a new team was created called Tools & Patterns. Their mission was to create shared libraries and template that developers in the org could use to stamp out products quickly, with prebuilt GitHub repositories and CICD pipelines. Things like CORS, Authn and Authz, containerization, and secret storage would all be created up front.

I joined this team after a couple of years on Orders, with the hopes of combining my experience "in the trenches" on a delivery team to solve problems for all of the delivery teams in the org. I work not only in front end and back end technologies, but in mentoring developers on patterns such as event sourcing, domain driven design, and command queueing.

Inergize Digital / Lakana // 2013 -> 2015

This was a mobile application development platform that could be used by news organizations owned by Nexstar Broadcasting to create and manage their mobile apps across iOS and Android. Users could create different sections in the application, and hook up content to the application via RSS. Changes to the app would automatically be published to Apple and Google. This provided a common interface for users.
